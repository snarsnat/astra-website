GitHub OAuth Setup for ASTRA

Follow these steps to connect your GitHub account to ASTRA for repository scanning and security analysis.

1

Register OAuth App

Create a new OAuth application in your GitHub Developer Settings.

Go to GitHub Developer Settings
2

Configure App Details

Fill in the application details with the information below.

3

Get Credentials

Copy your Client ID and Client Secret to configure ASTRA.

Application Details for GitHub OAuth

Use the production URL for your live app, or the local URL for development.

Configure ASTRA with Your Credentials

Environment Variables for Production (Vercel)

GITHUB_CLIENT_ID=your_client_id_here
GITHUB_CLIENT_SECRET=your_client_secret_here
GITHUB_CALLBACK_URL=https://astra-website-vercel-test.vercel.app/github-callback

Update server.py Configuration

# GitHub OAuth Configuration
GITHUB_CLIENT_ID = os.environ.get('GITHUB_CLIENT_ID', 'your_client_id_here')
GITHUB_CLIENT_SECRET = os.environ.get('GITHUB_CLIENT_SECRET', 'your_client_secret_here')
GITHUB_CALLBACK_URL = os.environ.get('GITHUB_CALLBACK_URL', 'http://localhost:3000/github-callback')

Required Permissions

  • repo - Full control of private repositories
  • read:user - Read user profile data
  • user:email - Read user email addresses

Test Your Connection

Once you've configured your GitHub OAuth app, test the connection:

Local Development Test

  1. Start the server: python server.py
  2. Open: http://localhost:3000/dashboard.html
  3. Click "Connect GitHub" button
  4. Authorize the application in GitHub
  5. You'll be redirected back to ASTRA

Production Test

  1. Deploy to Vercel with environment variables
  2. Open: https://astra-website-vercel-test.vercel.app/dashboard.html
  3. Click "Connect GitHub" button
  4. Authorize the application in GitHub
  5. You'll see your connected repositories
Go to Dashboard to Test Connection
